## 项目概述

### 这个项目是什么？
这是一个**自动化新闻收集工具**。它的工作就像一个"新闻助手"：
- 每天自动从互联网上的多个新闻源抓取 AI 相关的文章
- 整理这些文章（去掉重复的、过期的）
- 生成一份漂亮的日报（Markdown 格式）
- 保存到本地文件供你查看

### 为什么需要这个工具？
- **节省时间**：不用手动访问多个网站
- **集中管理**：所有新闻在一个文件里
- **自动化**：可以定时运行，每天自动更新
- **易于分享**：Markdown 格式可以轻松分享给他人

## 核心技术知识点

### 1. RSS 是什么？

**RSS** = Really Simple Syndication（真正简单的聚合）

#### 简单理解：
- 就像一个"新闻订阅"服务
- 网站把最新文章的信息放在一个特殊的文件里
- 我们的程序读取这个文件，获取最新文章

#### RSS 文件长什么样？
```xml
<?xml version="1.0"?>
<rss version="2.0">
  <channel>
    <title>TechCrunch AI</title>
    <item>
      <title>新文章标题</title>
      <link>https://example.com/article</link>
      <pubDate>Tue, 27 May 2026 14:30:00 GMT</pubDate>
      <description>文章摘要内容...</description>
    </item>
  </channel>
</rss>
```

#### 为什么用 RSS？
- ✅ 标准化格式，容易解析
- ✅ 网站自动更新，不需要手动爬虫
- ✅ 效率高，只获取必要信息
- ✅ 不会被网站反爬虫限制

### 2. 异步编程（Async/Await）

#### 问题：为什么需要异步？
假设我们要从 3 个网站获取数据，每个需要 2 秒：
- **同步方式**（一个接一个）：2 + 2 + 2 = 6 秒 ❌
- **异步方式**（同时进行）：最多 2 秒 ✅

#### 代码示例：
```typescript
// 同步方式 - 慢
const data1 = await fetchFeed1();  // 等待 2 秒
const data2 = await fetchFeed2();  // 等待 2 秒
const data3 = await fetchFeed3();  // 等待 2 秒
// 总共 6 秒

// 异步方式 - 快
const [data1, data2, data3] = await Promise.all([
  fetchFeed1(),
  fetchFeed2(),
  fetchFeed3()
]);
// 总共 2 秒（同时进行）
```

#### 我们的项目中：
```typescript
// 同时从 3 个源获取数据
const promises = sources.map(source => {
  if (source.type === 'hn') {
    return fetchHackerNews();
  }
  return fetchRssFeed(source.url, source.name);
});

const results = await Promise.all(promises);  // 同时执行
```

### 3. 数据去重（Deduplication）

#### 问题：为什么需要去重？
- 同一篇文章可能出现在多个新闻源
- 如果不去重，日报里会有重复内容

#### 解决方案：使用 Set（集合）
```typescript
const seen = new Set<string>();  // 存储已见过的 URL

for (const article of articles) {
  if (!seen.has(article.link)) {  // 如果没见过
    seen.add(article.link);        // 记录下来
    deduplicated.push(article);    // 加入结果
  }
}
```

#### 类比：
就像你在整理邮件时，看到重复的邮件就删除，只保留一份。

### 4. 时间处理（Date Filtering）

#### 问题：如何只获取最近 24 小时的文章？

```typescript
const now = new Date();                    // 当前时间
const twentyFourHoursAgo = subHours(now, 24);  // 24 小时前

// 只保留在这个时间范围内的文章
return articles.filter(article => 
  article.pubDate >= twentyFourHoursAgo
);
```

#### 为什么用 `date-fns` 库？
- 手动计算时间容易出错
- `date-fns` 提供了可靠的时间计算函数
- 自动处理时区问题

### 5. 类型系统（TypeScript）

#### 什么是 TypeScript？
- 是 JavaScript 的"升级版"
- 增加了**类型检查**功能
- 在代码运行前就能发现错误

#### 我们定义的类型：
```typescript
interface Article {
  title: string;        // 文章标题（必须是文本）
  link: string;         // 文章链接（必须是文本）
  pubDate: Date;        // 发布时间（必须是日期）
  source: string;       // 来源（必须是文本）
  description: string;  // 摘要（必须是文本）
}
```

#### 好处：
```typescript
// ✅ 正确
const article: Article = {
  title: "AI 新闻",
  link: "https://example.com",
  pubDate: new Date(),
  source: "TechCrunch",
  description: "摘要"
};

// ❌ 错误 - TypeScript 会提前警告
const article: Article = {
  title: 123,  // 错误！应该是文本，不是数字
  link: "https://example.com",
  pubDate: "2026-05-27",  // 错误！应该是 Date 对象
  source: "TechCrunch",
  description: "摘要"
};
```

#### 版本：26.2.0

#### 是什么？
- **包管理器** - 管理代码库和依赖
- 类似于 Python 的 pip 或 Ubuntu 的 apt

#### 主要功能：
```bash
npm install          # 安装项目依赖
npm start           # 运行项目
npm install package-name  # 安装新包
```

#### 我们的依赖：
```json
{

### 3. TypeScript

#### 是什么？
- JavaScript 的"超集"（包含 JavaScript 的所有功能，还有额外功能）
- 主要增加了**类型检查**

#### 为什么用 TypeScript？
| 特性 | JavaScript | TypeScript |
|------|-----------|-----------|
| 类型检查 | ❌ 运行时才发现错误 | ✅ 编写时就发现错误 |
| 代码提示 | ❌ 提示不完整 | ✅ 完整的代码提示 |

| 学习曲线 | ✅ 简单 | ⚠️ 稍微复杂 |

#### 编译过程：
```
TypeScript 代码 (.ts)
        ↓
   TypeScript 编译器
        ↓
JavaScript 代码 (.js)
        ↓

### 4. tsx

#### 是什么？
- **TypeScript 执行器**
- 让你直接运行 `.ts` 文件，不需要手动编译

#### 为什么需要？
```bash

```

### 5. rss-parser

#### 是什么？
- **RSS 解析库**
- 把 RSS 文件转换成 JavaScript 对象

#### 工作原理：
```typescript
import Parser from 'rss-parser';

const parser = new Parser();
const feed = await parser.parseURL('https://example.com/feed.xml');

// 自动转换为：
{
  title: "TechCrunch AI",
  items: [
    {
      title: "文章标题",
      link: "https://...",
      pubDate: "2026-05-27T14:30:00Z",
      description: "摘要..."
    }
  ]
}
```

#### 为什么用这个库？
- ✅ 自动处理 RSS/Atom 格式
- ✅ 自动解析日期
- ✅ 错误处理完善
- ✅ 不用自己写 XML 解析代码

### 6. date-fns

#### 是什么？
- **日期处理库**
- 提供各种日期操作函数

#### 我们用到的函数：
```typescript
import { subHours, format } from 'date-fns';

// 计算 24 小时前
const twentyFourHoursAgo = subHours(new Date(), 24);

// 格式化日期
const dateStr = format(new Date(), 'yyyy-MM-dd');
// 结果：2026-05-27
```

#### 为什么用这个库？
- ✅ 避免手动计算时间（容易出错）
- ✅ 自动处理时区
- ✅ 格式化日期简单
- ✅ 比 JavaScript 原生 Date 好用得多

## 常见问题

### Q1: 为什么有时候某个源会失败？

**A:** 可能的原因：
- 网络连接问题
- 网站服务器故障
- RSS 源地址变更
- 网站反爬虫限制

**我们的处理方式：**
```typescript
try {
  const feed = await parser.parseURL(url);
  return feed.items;
} catch (error) {
  console.warn(`⚠️  Failed to fetch ${source}`);
  return [];  // 返回空数组，继续处理其他源
}
```

### Q2: 为什么要去重？

**A:** 因为：
- 同一篇文章可能被多个新闻源转载
- 不去重的话，日报里会有重复内容
- 浪费阅读时间

### Q3: 为什么只保留 24 小时内的文章？

**A:** 因为：
- 日报是"每日"的，应该只包含最新内容
- 太旧的文章没有参考价值
- 保持日报的新鲜度

### Q4: 为什么用 TypeScript 而不是 JavaScript？

**A:** 
| 场景 | JavaScript | TypeScript |
|------|-----------|-----------|
| 小脚本 | ✅ 更快 | ⚠️ 有点重 |
| 中等项目 | ⚠️ 容易出错 | ✅ 更安全 |
| 大型项目 | ❌ 难维护 | ✅ 易维护 |

我们的项目虽然不大，但用 TypeScript 能：
- 防止类型错误
- 提供更好的代码提示
- 便于未来扩展

### Q5: 为什么用 RSS 而不是爬虫？

**A:**
| 方式 | RSS | 爬虫 |
|------|-----|------|
| 速度 | ✅ 快 | ❌ 慢 |
| 准确性 | ✅ 高 | ⚠️ 易变 |
| 反爬虫 | ✅ 无限制 | ❌ 易被限制 |
| 维护成本 | ✅ 低 | ❌ 高 |
| 合法性 | ✅ 合法 | ⚠️ 灰色 |

### Q6: 如何定时运行？

**A:** 可以用系统的任务计划程序：

**Windows:**
1. 打开"任务计划程序"
2. 创建基本任务
3. 设置触发器（每天某个时间）
4. 设置操作：运行 `run.bat`

**Linux/Mac:**
```bash

crontab -e

# 每天早上 8 点运行
0 8 * * * cd /path/to/project && node node_modules/tsx/dist/cli.mjs src/index.ts
```

### Q7: 如何修改新闻源？

**A:** 编辑 `src/index.ts` 中的 `FEED_SOURCES`：

```typescript
const FEED_SOURCES: FeedSource[] = [
  {
    name: '新源名称',
    url: 'https://example.com/feed.xml',
    type: 'rss',
  },
  // ... 其他源
];
```

### Q8: 如何修改摘要长度？

**A:** 编辑 `src/fetchers.ts` 中的 `extractSummary` 函数：

```typescript
function extractSummary(description: string | undefined): string {
  if (!description) return '';
  const cleaned = stripHtml(description);
  return cleaned.length > 100  // 改这个数字
    ? cleaned.substring(0, 100) + '...'
    : cleaned;
}
```
